# 多系列发布兼容设计

日期：2026-07-16

状态：approved for implementation

## 目标

让一个 `ai-work-system` 仓库承载多个文章系列，同时保证已经发布的“长期 AI 工作系统”15 篇文章继续正常通过 GitHub Wiki、Gitee Wiki 和墨问流水线。

## 兼容边界

- 现有 15 篇文章文件路径、Wiki 页面名称与 URL 不变。
- Wiki 页面仍使用全局编号稳定 Gitee 页面顺序；系列内导航使用独立序号。
- 第一系列文章的墨问标题继续为 `01-` 至 `15-`。
- 旧 `publishing/mowen-notes.json` 的单例 `directory` 自动迁移到第一系列，不丢失 note ID、封面 UUID、内容哈希和发布状态。
- 新系列不会进入第一系列的 Wiki 连续阅读链或墨问目录。

## 结构设计

新增 `content/series.json` 作为系列元数据源，保存：

- 稳定 series ID；
- 展示顺序；
- 中英文标题与说明；
- 生命周期状态；
- Wiki 系列目录页名称；
- 墨问目录标题、介绍和公开 URL。

文章继续保存在 `content/articles/*.md`，并通过现有 `series` frontmatter 关联系列，不移动现有源文件。

## README

README 的自动生成区块按系列分组：

- 外层标题改为“系列文章 / Article Series”；
- 每个已有 `ready` 文章的系列生成独立小节；
- 显示系列状态、文章数量和系列专属墨问目录；
- Wiki 链接继续指向现有全局编号页面。

## Wiki

- `Home.md` 按系列展示文章。
- 每个系列生成独立目录页。
- `_Sidebar.md` 使用系列和文章两层结构。
- 文章的上一篇和下一篇只在当前系列内计算。
- 文章底部“目录”链接到当前系列目录页。
- 现有页面继续使用 `01-` 至 `15-`；新文章继续使用全局编号，避免 Gitee 排序回退和旧 URL 失效。

## 墨问

映射格式升级为：

```json
{
  "version": 2,
  "directories": {
    "long-term-ai-work-system": {}
  },
  "articles": {}
}
```

每个系列：

- 独立计算 `01-` 开始的标题编号；
- 独立构建倒序文章目录；
- 独立保存目录 note ID、URL、封面和内容哈希；
- 只嵌入本系列文章。

第一版保留现有 `--cover-url` 作为第一系列封面参数。第二系列目录和封面由 Lee 创建或确认后，再登记到 `directories.<series-id>`，不在本次兼容改造中自动创建公开内容。

## 验收

- 现有 15 篇 Wiki 页面名称与改造前完全一致。
- 第一系列首尾导航仍为“无”，第 15 篇不会跳到第二系列。
- Wiki Home、Sidebar 和系列目录页均按系列分组。
- 第一系列墨问文章编号与旧版一致，旧目录映射无损迁移。
- 两个平台发布前验证、Wiki 单元测试、墨问单元测试和 README 生成测试全部通过。
