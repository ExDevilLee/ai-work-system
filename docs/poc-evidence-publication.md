# POC 公开依据规范

本文定义实战文章所依赖 POC 的公开结构。目标是在保留可复查证据的同时，避免把大量重复运行目录直接提交给读者。

## 三层证据

### 完整原始记录

完整事件流、stderr、本机环境和失败上下文写入 POC 的 `runs/private/`，只在本地保留并由 Git 忽略。

### 完整脱敏中间层

每次运行先导出到 `runs/public/`，删除本机路径、会话标识、密钥和无关全局上下文。该目录用于本地审计和生成公开包，同样由 Git 忽略。

### 提交到仓库的公开包

最终提交 `evidence/`：

```text
evidence/
├── README.md
├── manifest.jsonl
├── fixtures/
└── representative-runs/
```

- `manifest.jsonl` 记录全部运行的任务、条件、模型、推理强度、执行器版本、评分、指标和校验和。
- `representative-runs/` 默认展开一轮完整正式对照、关键失败和必要的模型差异样本，通常控制在 6 至 10 个目录。
- 相同夹具和提示只保留一份，代表运行通过相对路径和 SHA-256 引用，不在每个目录重复复制。
- 未被展开的运行仍保留在 manifest 和汇总 CSV/JSON 中，不因压缩目录而丢失实验结果。

## 代表样本选择

代表样本优先保留：

1. 覆盖所有任务和实验条件的一轮正式运行。
2. 促使协议、脚本或判断发生修正的真实失败。
3. 能说明模型工具策略差异的最小对照。
4. 异常值或测量修正只有在帮助读者理解限制时才展开，否则写入报告和 manifest。

不要只选择支持预期结论的成功样本，也不要为了“证据完整”展开所有高度重复目录。

## 模型记录

每次正式实验至少记录：

- 模型名称。
- 推理强度。
- 执行器及版本。

不同模型和推理强度单独汇总。文章中的主结论使用固定模型配置，其他模型只作为独立敏感性对照，不混合计算。

## 文章链接

实战文章设置“实验与复现”章节，链接到对应 POC 根目录。读者应该能从 POC 根目录找到：

- 实验协议与适用范围。
- 夹具、提示和复现命令。
- 正式数据与分析报告。
- 精简公开依据和全部运行 manifest。
- 失败、限制和仍未验证的平台。

文章不直接链接 `runs/private/` 或本地脱敏中间层。

## 提交门禁

提交前至少验证：

1. manifest 中运行数量与本地脱敏记录一致，run name 唯一。
2. 代表样本的夹具和提示校验和匹配。
3. 所有代表样本链接可从仓库内解析。
4. 公开包不包含本机绝对路径、会话标识、密钥和用户数据。
5. 公开报告中的模型、样本数量和指标可以从 manifest 与汇总数据复算。
6. Git 不会纳入 `runs/private/` 和 `runs/public/`。
