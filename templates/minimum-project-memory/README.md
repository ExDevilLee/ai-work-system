# 最小项目记忆模板

这套模板用于给一个需要长期与 AI 协作的项目建立最小上下文。它不要求知识库或复杂工具，只先回答三个问题：

1. 这个项目是什么？
2. AI 应该按什么规则继续？
3. 项目现在进行到哪一步？

## 目录结构

```text
project/
├── README.md
├── AGENTS.md
└── PROJECT-STATE.md
```

- `README.md`：面向人和 AI 的项目入口，记录目标、范围、内容源头和常用入口。
- `AGENTS.md`：面向 AI 的协作规则，记录读取顺序、修改边界、验证方式和信息安全要求。
- `PROJECT-STATE.md`：面向当前工作的状态入口，记录目标、进展、下一步、阻塞和近期决策。

三个文件各自只回答一类问题：README 说明“这是什么”，AGENTS 说明“应该怎么做”，PROJECT-STATE 说明“现在做到哪里”。

## 使用方法

1. 将 [`project/`](project/) 中的三个文件复制到你的项目根目录。
2. 搜索 `{...}` 形式的占位内容，替换成项目的真实信息。
3. 删除不适用的示例，不要为了显得完整而保留空规则。
4. 让 AI 每次进入项目时按 `README.md`、`AGENTS.md`、`PROJECT-STATE.md` 的顺序读取。
5. 完成一个阶段后更新 `PROJECT-STATE.md`；只有经过验证、未来会重复影响工作的经验，才写入 `AGENTS.md`。

可以用下面的命令检查是否还有未替换的占位内容：

```bash
rg -n '\{[^}]+\}' README.md AGENTS.md PROJECT-STATE.md
```

## 维护边界

- 一个事实只保留一个可信源头，其他文件用链接指向它。
- 临时进展进入 `PROJECT-STATE.md`，稳定规则进入 `AGENTS.md`。
- 不把密码、token、cookie、私钥或客户敏感信息写进这些文件。
- 不要保存所有聊天记录；优先保存已经验证、未来可复用的结论。
- 当文件开始变长时，先清理过期内容，再考虑增加新的记忆工具。

## 什么时候继续扩展

只有当这三个文件已经真实使用，并且仍然反复出现同一类问题时，再考虑增加任务台账、操作指南、自动检查脚本或知识索引。扩展的目标应该是减少重复解释，而不是让系统看起来更复杂。
