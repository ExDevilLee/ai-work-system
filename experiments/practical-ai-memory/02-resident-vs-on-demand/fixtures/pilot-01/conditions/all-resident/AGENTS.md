## 高后果稳定规则

- 公开文章中的实践结论必须来自已经运行的真实实验。
- 平台兼容性结论只能覆盖已经完成真实验证的平台。
- 完整原始运行证据只保留在本地私有区，不进入公开仓库。

## 低频稳定规则

- 新文章进入 review 前，必须回看上一篇结尾钩子是否冲突。
- 配图只有在能够明显降低理解成本时才添加。

## 待验证观察

- 选择性常驻可能比全部常驻减少无关上下文。
- 仅保留索引可能增加高后果规则的检索成本。

## 已失效结论

## 公开证据包检查

公开 POC 前必须确认：

1. 代表样本能够追溯到完整 manifest。
2. 本机绝对路径、会话标识和疑似密钥已经移除。
3. README 提供最小复现命令和实验限制。

## 平台声明策略

- macOS 正式矩阵通过后，只能声明已在 macOS 实测。
- Win11 复现完成并通过相同门禁后，才能声明已在 macOS 与 Win11 实测。
- 只完成代码兼容性检查时，应写“理论兼容”或“待验证”，不能写“实测通过”。

## 文章 Review 检查

文章进入 review 前核对：

1. 结论能够追溯到实际运行证据。
2. 上一篇结尾钩子与当前主题不存在冲突。
3. 配图能够降低理解成本，而不是只用于装饰。
