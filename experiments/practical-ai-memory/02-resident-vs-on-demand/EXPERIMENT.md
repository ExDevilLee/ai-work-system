# 实验协议

## 研究问题

在项目事实和文件结构完全相同的前提下，改变自动加载入口中的常驻信息量，是否会影响 Agent 找到必要事实的正确性、项目上下文读取量、无关信息带入和人工 Review 成本？

## 待验证假设

选择性常驻可能在两类风险之间取得平衡：

- 相比全部常驻，减少与当前任务无关的自动注入内容。
- 相比仅保留索引，降低高后果稳定规则被遗漏的风险和检索成本。

这只是实验假设。即使选择性常驻在当前夹具中表现更好，也不能推出所有项目都应使用同一套文件或固定字节阈值。

## 实验变量

三组使用相同的 `common/` 事实文件和任务提示，只替换根目录 `AGENTS.md`。

### All Resident：全部常驻

`AGENTS.md` 复制全部测试事实。Agent 可以不读取其他文件就回答所有任务，但每个会话都会自动携带与当前问题无关的内容。

### Selective Resident：选择性常驻

`AGENTS.md` 只复制三条满足以下条件的规则：

1. 已经稳定确认，而不是当前状态或待验证观察。
2. 跨多类任务重复适用。
3. 遗漏后可能造成公开失实、越界披露或错误平台声明。
4. 内容足够短，适合持续维护。

当前状态、低频规则、观察、失效结论和参考资料只保留导航入口。

### Index Only：仅索引

`AGENTS.md` 只说明各类信息位于哪里，不复制任何测试事实。每个任务都需要按需读取相应文件。

## 等价性约束

- 三组的 `memory/` 和 `references/` 来自同一个共享目录。
- `expected/facts.json` 中每条事实必须在共享文件里出现且只出现一次。
- 三组提示完全相同，不暴露目标文件名或条件名称。
- `all-resident` 必须包含全部事实；`selective-resident` 只能包含标记为常驻的事实；`index-only` 不得包含答案事实。
- `validate_fixtures.py` 未通过时，不得运行模型。

## 测试任务

每个任务使用独立的新会话，避免前一任务的读取结果进入后一任务。

| 任务 | 主要验证目标 | 预期按需来源 |
| --- | --- | --- |
| `critical-boundary` | 高后果稳定规则能否被正确应用 | `memory/MEMORY.md` |
| `reference-detail` | 低频细节能否通过索引准确找到 | `references/INDEX.md` 与目标资料 |
| `volatile-state` | 易变化的当前状态是否需要常驻 | `memory/CURRENT.md` |
| `status-conflict` | 观察和失效结论是否被误当成事实 | `memory/MEMORY.md` |

## 指标

### 正确性与质量

- 每项预期事实是否命中。
- 事实在共享项目记录中的原始来源路径是否正确。
- 是否出现无依据断言。
- 是否把待验证观察或失效结论写成当前事实。
- 是否加入问题未要求的无关项目事实。

### 上下文与执行

- `resident_instruction_bytes`：根目录 `AGENTS.md` 的 UTF-8 字节数。
- `workspace_output_bytes`：可可靠识别的工作区工具输出字节数。
- `project_context_bytes`：上述两项之和，只在工作区指标覆盖完整时使用。
- 工作区调用次数、耗时和 token。

`project_context_bytes` 是项目侧上下文的近似量，不等于模型计费 token。token、耗时和调用次数只作为过程指标，不能单独代表交付质量。

### 人工成本

Pilot 期间记录：

- 是否能按冻结答案确定评分。
- 是否需要人工解释来源或修正分类。
- 每个答案的 Review 分钟数。

## Pilot 01 矩阵

首次试跑为 `4 个任务 x 3 个条件 x 1 次`，共 12 次。条件顺序按任务轮换，降低固定先后顺序影响：

1. `critical-boundary`：all、selective、index。
2. `reference-detail`：selective、index、all。
3. `volatile-state`：index、all、selective。
4. `status-conflict`：all、index、selective。

Pilot 只验证协议，不进入文章数据。出现以下任一情况时停止并修订：

- 条件间共享事实不等价。
- 提示直接泄漏文件路径或加载策略。
- 冻结答案无法确定评分。
- 自动注入的 `AGENTS.md` 未被执行器实际加载。
- 工作区读取方式超出指标审计器覆盖范围。
- 某个条件因权限、编码或运行环境失败。

### Pilot 01 结果与修订

12 次运行全部完成，三组均正确回答四类任务，指标覆盖完整。Pilot 同时发现 `critical-boundary` 把规则决策和原始来源审计放在同一个问题中，会迫使 Agent 为核对来源继续读取文件，无法单独观察高后果规则常驻后的决策路径。

Pilot 02 只修订并重跑该任务：保留两项违规场景、规则判断和修正要求，移除来源路径要求。其他三个任务及夹具不变，不重复运行。逐任务评分项在 `expected/rubric.json` 中冻结；Pilot 01 的完整复核见 [`analysis/pilot-01.md`](analysis/pilot-01.md)。

### Pilot 02 结果与修订

三组均获得 6/6，指标覆盖完整。但 `selective-resident` 和 `index-only` 的 `AGENTS.md` 仍无条件要求答案给出来源，导致题目虽然删除来源审计，Agent 仍主动读取文件。Pilot 02 因此不能用于比较纯决策路径。

Pilot 03 将两组导航规则改为“只有题目要求来源时才给出原始来源路径”，再次只重跑 `critical-boundary`。事实、提示和评分表不变。完整复核见 [`analysis/pilot-02.md`](analysis/pilot-02.md)。

### Pilot 03 结果

三组再次获得 6/6，指标覆盖完整。仅索引读取当前状态和稳定记忆；选择性常驻仍自主核对稳定记忆与两份参考资料；全部常驻没有工作区读取。此时已经不存在无条件来源审计要求，因此工具路径差异作为真实行为进入正式重复实验，不再继续调整提示。

Pilot 03 通过协议门禁。正式矩阵固定为 `4 个任务 x 3 个条件 x 3 次`，共 36 次，逐任务评分表、模型、推理强度、夹具和提示均冻结。完整复核见 [`analysis/pilot-03.md`](analysis/pilot-03.md)。

## 正式实验门槛

正式矩阵已经冻结。每个任务与条件重复 3 次，并交替条件顺序。正式运行固定并记录模型、推理强度、Codex CLI、平台、夹具哈希和提示哈希。

macOS 是首轮主实验平台。Win11 使用同一冻结协议进行后续复现；在 Win11 完成前，文章不得声称已经跨平台验证。

## 有效性限制

- `AGENTS.md` 是 Codex App 和 CLI 的自动加载入口，其他 Agent 工具可能使用不同规则文件。
- 夹具规模远小于真实长期项目，结果只能支持相对趋势和失败模式。
- 常驻规则的选择包含人工判断，实验不能证明存在适用于所有项目的统一分类器。
- 自动注入内容与工具读取内容通过不同路径进入模型，字节相加只是透明的近似指标。
