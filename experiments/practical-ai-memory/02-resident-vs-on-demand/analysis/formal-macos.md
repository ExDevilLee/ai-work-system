# macOS 正式矩阵分析

## 实验范围

本轮使用冻结协议完成 `4 个任务 x 3 个条件 x 3 次`，共 36 次正式运行：

- 平台：macOS
- 模型：`gpt-5.6-sol`
- 推理强度：`medium`
- Codex CLI：`0.144.1`
- 三个条件：`all-resident`、`selective-resident`、`index-only`

36 个样本均正常退出，工作区指标覆盖完整，输出字节可可靠统计。人工 Review 按任务批量计时并均摊到样本，因此该时间只能说明本轮评分成本，不能用于细粒度比较条件差异。

完整聚合数据：

- [`formal-macos-gpt-5.6-sol-medium.json`](../data/formal-macos-gpt-5.6-sol-medium.json)

## 正确性

三种策略均获得满分：每个条件 12 个样本，合计 `60/60`。36 个答案均未发现无依据断言或问题未要求的项目事实。

这说明在当前小型、结构清晰且索引有效的夹具中，三种加载策略都能正确恢复所需事实。它不说明三种策略在真实大型项目中具有相同可靠性。

## 总体过程指标

以下均为每个条件 12 次运行的统计值。项目上下文字节为 `resident_instruction_bytes + workspace_output_bytes`，只表示项目侧上下文近似量，不等于计费 token。

| 条件 | 常驻字节 | 项目上下文字节中位数 | 工作区调用中位数 | 耗时中位数（秒） |
| --- | ---: | ---: | ---: | ---: |
| 全部常驻 | 2438 | 2438 | 0 | 18.665 |
| 选择性常驻 | 812 | 2554.5 | 1.5 | 37.775 |
| 仅索引 | 477 | 1727.5 | 2 | 28.983 |

全部常驻在本轮静态夹具中省去了大多数文件读取，因此耗时最短；代价是每个任务固定注入全部 2438 字节，包括与当前任务无关的事实。

选择性常驻没有在总体项目上下文字节上胜出。模型即使已经收到高后果规则，仍会在部分任务中主动核对共享记录，形成“常驻内容 + 文件读取”的叠加成本。这是正式矩阵中的真实行为，不应通过继续调整提示来消除。

## 分任务结果

表中为项目上下文字节中位数。

| 任务 | 全部常驻 | 选择性常驻 | 仅索引 | 当前样本中的最低值 |
| --- | ---: | ---: | ---: | --- |
| 高后果边界 | 2438 | 2954 | 1744 | 仅索引 |
| 低频参考细节 | 2438 | 1655 | 1260 | 仅索引 |
| 易变当前状态 | 2438 | 1331 | 996 | 仅索引 |
| 状态冲突分类 | 2438 | 4105 | 3536 | 全部常驻 |

差异不是简单的“常驻越少越好”：

- 低频资料和易变状态能够通过明确索引直接定位，按需读取的上下文更小。
- 状态冲突需要同时理解观察、失效结论和证据边界，按需组会读取更多上下文。
- 高后果规则即使已经选择性常驻，模型仍可能为了确认边界继续读取文件；常驻不能保证省去检索。
- 全部常驻在静态夹具中看起来执行成本低，但实验没有模拟状态过期、入口膨胀、规则互相冲突和长期维护成本。

## 当前可以支持的结论

本轮结果支持任务依赖的分层策略，而不支持一个固定字节阈值或“选择性常驻总是最优”的结论：

1. 稳定、跨任务、高后果且足够短的规则，适合作为常驻候选，主要价值是降低遗漏风险，而不是保证减少读取。
2. 易变化的当前状态和低频参考细节适合按需读取，前提是入口索引能让 Agent 准确定位。
3. 待验证观察和已失效结论应保留在可检索记录中，并明确状态；是否常驻应根据冲突风险单独判断。
4. 全部常驻能减少当前静态任务的工具读取，但会持续携带无关内容，并引入本实验尚未覆盖的更新一致性风险。

因此，选择信息放置位置时应先看遗漏后果、变化频率、跨任务复用和维护风险，再看单次运行的 token、耗时或工具调用数。

## 运行异常

正式矩阵期间有一次模型服务在生成答案前返回 nginx `400 Bad Request`。失败记录已隔离，未进入 36 个正式样本；随后使用相同提交、模型、提示和参数重跑成功。该事件属于服务可用性异常，不作为条件差异解释。

## 有效性限制与下一步

- 每个任务只有 3 次重复，样本不足以支持稳定的耗时或 token 排名。
- 夹具很小，且所有事实在运行期间保持同步，未覆盖真实长期项目的陈旧状态和维护成本。
- 人工 Review 未盲化条件名称，且采用批次均摊时间，不能用于细粒度条件比较。
- 结果只覆盖 macOS、当前模型与当前 Codex CLI；Win11 复现完成前不得声称跨平台实测。
- 公开前仍需构建脱敏证据包，移除绝对路径、会话标识和私有原始事件。
