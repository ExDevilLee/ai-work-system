# macOS 正式矩阵分析

## 实验范围

本轮使用冻结协议完成 `4 个任务 x 3 个条件 x 3 次`，共 36 次正式运行：

- 平台：macOS
- 模型：`gpt-5.6-sol`
- 推理强度：`medium`
- Codex CLI：`0.144.1`
- 三个条件：`all-resident`、`selective-resident`、`index-only`
- 插件：显式关闭

36 个样本均正常退出，`runtime_tool_access_calls=0`，工作区指标覆盖完整，输出字节可可靠统计。

## 为什么重新运行

首次 Win11 Smoke 暴露了用户级 Codex 插件可能进入实验上下文。加入运行时越界门禁后回扫旧 macOS 正式记录，发现 12/36 次也读取了插件技能文件。

旧审计器只把这些读取排除在项目上下文指标之外，没有把它们判为协议失败。按 Win11 使用的同一标准，旧 macOS 矩阵不能继续作为干净基线，因此 36 次全部隔离并重新运行。旧聚合数据和旧代表样本不再用于文章结论。

| 条件 | 得分 | 扣分原因 |
| --- | ---: | --- |
| 全部常驻 | 60/60 | 无 |

36 个答案均未发现无依据断言或问题未要求的项目事实。仅索引的扣分属于有效遗漏，不是协议失败，也没有通过补跑消除。

## 总体过程指标

以下均为每个条件 12 次运行的中位数。项目上下文字节为 `resident_instruction_bytes + workspace_output_bytes`，只表示项目侧上下文近似量，不等于计费 token。

| 条件 | 常驻字节 | 项目上下文字节 | 工作区调用 | 耗时（秒） |
| --- | ---: | ---: | ---: | ---: |
| 全部常驻 | 2,438 | 2,438 | 0 | 23.554 |
| 选择性常驻 | 812 | 3,149 | 3 | 38.536 |
| 仅索引 | 477 | 1,425 | 1 | 31.670 |

全部常驻在当前静态夹具中不需要读取项目文件。仅索引使用最少项目上下文，但在高后果任务中三次遗漏同一个修正项。选择性常驻保持满分，却没有减少总体项目上下文。

## 分任务结果

表中为项目上下文字节中位数。

| 任务 | 全部常驻 | 选择性常驻 | 仅索引 |
| --- | ---: | ---: | ---: |
| 高后果边界 | 2,438 | 3,587 | 1,711 |
| 低频参考细节 | 2,438 | 2,132 | 1,090 |
| 易变当前状态 | 2,438 | 1,331 | 996 |
| 状态冲突分类 | 2,438 | 4,087 | 1,711 |

macOS 结果支持“仅索引通常减少当前项目上下文”，但不支持把它写成无损优化。高后果任务中的稳定遗漏说明，导航可达不等于答案一定完整。

## 当前可以支持的结论

1. 全部常驻能消除当前静态夹具中的项目文件读取，但实验尚未覆盖入口膨胀、陈旧副本和长期维护成本。
2. 仅索引在 macOS 上使用最少项目上下文，同时出现稳定的高后果修正项遗漏。
3. 选择性常驻保持正确性满分，但不能保证减少读取或上下文。
4. 加载策略需要同时权衡变化频率、遗漏后果、检索可靠性和维护风险。

## 有效性限制

- 每个任务与条件只有 3 次重复，不支持稳定的耗时或 token 排名。
- 夹具远小于真实长期项目，所有事实在运行期间保持静态且同步。
- Review 未盲化条件名称，并采用按任务批量计时。
- 结果只覆盖当前模型、推理强度、CLI 和 macOS 工具链。
- 跨平台解释见 [`formal-macos-win11-comparison.md`](formal-macos-win11-comparison.md)。
