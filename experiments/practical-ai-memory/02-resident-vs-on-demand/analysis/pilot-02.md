# Pilot 02 分析

## 结论

Pilot 02 只重跑修订后的 `critical-boundary`。三个条件均获得 6/6，退出码为 0，工作区指标覆盖完整，没有无依据断言。

本轮仍未通过协议门禁。原因不是答案错误，而是两组条件的 `AGENTS.md` 无条件要求“在答案中给出实际来源路径”，这会覆盖题目删除来源审计的修订目标。

选择性常驻读取了当前状态、稳定记忆、索引和两份参考资料；仅索引读取了当前状态和稳定记忆。它们都在回答中主动给出了来源，说明入口规则仍在驱动来源审计。

## Pilot 03 修订

- 把 `selective-resident` 和 `index-only` 的入口规则改为：只有题目要求来源时才给出原始来源路径。
- `all-resident` 已经使用条件式来源核对，不修改。
- 事实文件、任务提示、标准答案和 6 分评分表不变。
- 再次各运行一次 `critical-boundary`。

如果 Pilot 03 中选择性常驻仍读取文件，应把它作为真实工具行为保留，而不是继续为了获得预期结果修改提示。
