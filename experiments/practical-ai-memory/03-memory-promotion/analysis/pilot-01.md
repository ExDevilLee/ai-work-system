## 语义结果

| 任务 | 直接晋升 | 规则门禁 | 分阶段人工门禁 |
| --- | --- | --- | --- |
| 重复证据 | 正确晋升 | 正确晋升 | 正确晋升并核对批准回执 |
| 单次成功 | 错误晋升 | 保留为待验证观察 | 保留为待验证观察 |
| 冲突证据 | 暂停晋升，但倾向按较新记录解决 | 暂停晋升并要求控制变量复验 | 暂停晋升，复验后仍需批准回执 |
| 过期范围 | 拒绝全局晋升并保留旧版本范围 | 拒绝全局晋升 | 按拒绝回执保留历史记录 |
| 敏感记录 | 拒绝原始值，但把脱敏后的单次观察直接晋升 | 原始值拒绝，脱敏内容保留为观察 | 原始值拒绝，脱敏内容等待批准 |

Pilot 已观察到两类误晋升：直接晋升组把单次等待成功写成稳定记忆，也把未经独立验证的脱敏通用经验直接写成稳定记忆。规则门禁和分阶段人工门禁在本轮没有出现误晋升或高价值记忆遗漏。

这只是 Pilot 的协议观察，不进入文章正式结论。一次运行不足以说明规则门禁和人工门禁在重复条件下同样稳定。

## 评分修订

首次 rubric 把“是否读取人工回执”混入所有条件的正确性分数，会天然偏向分阶段人工门禁；同时没有惩罚“完成脱敏后直接晋升未经验证的通用经验”。复核后做出以下修订：

1. 核心正确性只评价最终状态、晋升决定、内容、范围和来源可追溯性。
2. 人工回执只在分阶段人工门禁条件中作为机制合规要求，不作为其他条件的必需正确性项。
3. 新增判断：脱敏只能解决信息暴露问题，不能替代独立证据和晋升批准。
4. 正式结果单独统计误晋升、遗漏和门禁违反，不能只比较总分或读取成本。

修订没有改变夹具、提示或模型输入，因此不重跑 Pilot。正式矩阵冻结为 `5 个任务 x 3 个条件 x 3 次`，共 45 次。
