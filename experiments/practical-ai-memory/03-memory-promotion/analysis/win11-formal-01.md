# Win11 Formal 01：HTTP 429 后的安全恢复

Win11 完整 Smoke 通过后启动 45 次正式矩阵。前 32 次成功，第 33 次在两次 Windows 沙箱拒绝后恢复读取，但随后遇到 HTTP 429，重试耗尽并以退出码 1 停止；剩余 12 次没有启动。

失败运行没有最终答案和用量记录，不属于有效正式样本。它的环境隔离和工作区指标覆盖仍然有效，但不能因为存在 `metadata.json` 就视为完成。

## 暴露的恢复缺口

矩阵调度器原本只检查 `metadata.json` 是否存在。实验执行器无论成功或失败都会写 metadata，因此直接恢复可能跳过失败槽位，把 44 个有效样本误报成完整的 45 次矩阵。

## 修正

Pilot 与 Formal 调度器现在共享同一完成性门禁。只有同时满足下面条件的运行才允许在恢复时显示 `SKIP`：

- `exit_code=0`
- 最终答案非空
- 用量记录存在
- 环境隔离有效
- 工作区指标覆盖完整且输出可靠
- metadata、final、raw、stderr、prompt 和夹具快照完整

失败目录必须先隔离并保留原因。外部限流冷却后，可以使用相同标签、任务、条件、模型和推理强度重跑失败槽位，再由矩阵脚本跳过 33 个有效样本并继续剩余 12 次。已经成功的 32 次不需要重跑。

如果再次出现 429，应继续停止并延长冷却时间，不通过删除有效样本、替换任务或增加额外重复来掩盖失败。
