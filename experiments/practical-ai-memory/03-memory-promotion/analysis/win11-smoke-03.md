# Win11 Smoke 03：临时路径链接不是协议失败

第三批 Win11 Smoke 在第 1 次运行后被隔离，原因是最终答案把夹具来源渲染成临时工作区的绝对路径链接。复核后确认，这条停止条件过严。

## 边界判断

实验运行本来就使用独立临时工作区，私有 `final.md` 和 `raw.jsonl` 也不会直接公开。只要链接满足下面条件，就不影响实验隔离有效性：

- 路径严格位于本次临时夹具目录。
- 没有读取真实用户目录、真实记忆、用户级插件或外部项目文件。
- 运行时访问审计与指标覆盖门禁仍然通过。

这种链接仍然包含设备相关路径，不能直接进入公开证据。公开导出时必须删除绝对路径、改为工作区相对来源标签，或者不公开对应原始答案。

## 后续处理

第三批第 1 次运行已经隔离，不再复用。下一批需要从头执行 15 次 Smoke，但不再因为隔离临时路径链接单独判定协议失败。真实用户路径、隔离目录外访问、插件读取、指标漏记和语义答案异常仍然是停止条件。
