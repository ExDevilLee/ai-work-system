# Win11 Smoke 01：MCP 片段读取漏记

首次 Win11 Smoke 在第 1 次运行后停止，没有执行其余 14 次 Smoke 或 45 次正式矩阵。

## 现象

- PowerShell 工作区读取被 Windows 沙箱拒绝。
- 模型随后通过 `node_repl` MCP 调用恢复读取并正确回答。
- 两次 MCP 结果明确包含夹具内容片段，但指标分类器没有把它们计为工作区读取。
- 元数据因此错误地显示工作区 MCP 调用为 0，同时把指标覆盖标记为完整。

答案正确不能弥补过程指标漏记。该运行已经隔离，不进入有效 Smoke 或正式结论。

## 根因

分类器原本支持整份夹具内容和 JSON 包装的多文件结果，但要求完整文件原文出现在 MCP 结果中。本次 `node_repl` 只返回文件内容片段，因此被静默归类为非工作区调用。

## 修正

分类器保留整文件精确匹配，并增加足够长的夹具内容片段匹配。短标签或普通词重合不计为文件读取，避免把无关 MCP 输出误判为工作区内容。

新增回归测试覆盖：

- 结构化 MCP 结果中的长夹具片段应计入工作区调用和输出字节。
- 短文本与夹具重合时不得误判。

修正后必须从头执行完整的 15 次 Win11 Smoke。首次隔离运行不能复用；只有新 Smoke 的环境隔离、语义答案和指标覆盖全部通过后，才能启动正式矩阵。
