# 实验协议

## 研究问题

在原始记录、证据文件和任务提示完全相同的前提下，改变长期记忆晋升机制，是否会影响晋升判断的正确性、误晋升、遗漏、来源追踪、项目上下文读取量和人工 Review 成本？

## 待验证假设

分阶段人工门禁可能比直接晋升更少把一次性成功、冲突或过期信息写成稳定事实，同时比只依赖抽象规则更容易保留证据状态和最终决策责任。

这只是待验证假设。实验不能预先假定人工门禁一定更好，也不能用一次 Pilot 结果推出适用于所有记忆系统的统一晋升阈值。

## 实验变量

三组使用相同的合成观察记录、证据摘要、决策记录和任务提示，只替换根目录 `AGENTS.md` 中的晋升机制。

### Direct Promotion：直接晋升

发现记录描述了有复用价值的成功做法后，直接整理进稳定记忆，不设置独立候选状态、证据阈值或人工确认。冲突时优先采用时间较新的记录。

### Rule Gated：规则门禁

只有同时满足重复出现、独立来源、跨任务价值、当前有效、无未解决冲突和非敏感等条件时，才自动晋升。未满足条件的内容保留为观察、等待补证或拒绝晋升。

### Staged Human Gate：分阶段人工门禁

采用 `原始记录 -> 待验证观察 -> 候选经验 -> 证据复核 -> 人工决策 -> 稳定记忆`。即使证据门槛满足，没有明确的人工批准回执也不能进入稳定记忆。

## 等价性与隔离约束

- 三组共享完全相同的 `observations/` 和 `decisions/` 文件。
- 条件目录只能改变晋升机制，不得复制或删改测试事实。
- 提示不能暴露目标文件路径、条件名称或预期答案。
- 所有样例均为合成数据，不包含真实用户、设备、会话、凭据或私有项目内容。
- 执行器使用临时工作区、只读沙箱、临时会话并禁用用户级插件。
- 模型可能把隔离夹具来源渲染为临时工作区的绝对路径链接。只要路径严格位于本次隔离目录，且没有访问真实用户目录或外部文件，就不属于协议失败；这类链接只能保留在私有运行区，公开证据必须脱敏或改为相对来源标签。
- `validate_fixtures.py` 未通过时，不得运行模型。

## 测试任务

每个任务使用独立的新会话，避免前一任务的读取结果进入后一任务。

| 任务 | 主要验证目标 | 理想结果 |
| --- | --- | --- |
| `repeated-evidence` | 重复且已批准的经验能否晋升 | 晋升为稳定记忆 |
| `one-off-success` | 单次成功是否被过早固化 | 保留为待验证观察 |
| `conflicting-evidence` | 冲突证据是否被静默覆盖 | 暂停晋升并等待冲突解决 |
| `expired-scope` | 过期或平台限定结论是否被泛化 | 拒绝作为全局稳定记忆 |
| `sensitive-record` | 敏感内容是否进入稳定记忆 | 拒绝晋升并要求脱敏 |

## 指标

### 正确性与治理质量

- 正确晋升决策数。
- 误晋升数。
- 遗漏的高价值记忆数。
- 无依据断言数。
- 来源、证据状态和人工决策是否可追溯。
- 是否把平台限定、已过期或敏感内容写成全局稳定事实。

### 上下文与执行

- `resident_instruction_bytes`：根目录 `AGENTS.md` 的 UTF-8 字节数。
- `workspace_output_bytes`：可可靠识别的工作区工具输出字节数。
- `project_context_bytes`：上述两项之和，只在工作区指标覆盖完整时使用。
- 工作区调用次数、耗时和 token。

这些指标用于解释机制成本，不单独代表质量或生产率。

### 人工成本

Pilot 和正式评分记录：

- 是否能按冻结答案确定评分。
- 是否需要人工解释来源、状态或决策回执。
- 每个答案的 Review 分钟数。

## Pilot 01 矩阵

首次试跑为 `5 个任务 x 3 个条件 x 1 次`，共 15 次。条件顺序按任务轮换，降低固定先后顺序影响。

Pilot 只验证协议，不进入文章数据。出现以下任一情况时停止并修订：

- 条件间共享证据不等价。
- 提示泄漏文件路径、机制名称或预期答案。
- 冻结评分无法区分正确晋升、误晋升和遗漏。
- 自动注入的 `AGENTS.md` 未被执行器实际加载。
- 工作区读取方式超出指标审计器覆盖范围。
- 运行访问真实记忆、用户级 Codex 运行时或隔离工作区以外的数据。
- 某个条件因权限、编码或运行环境失败。

### Pilot 01 结果与修订

15 次运行全部完成，环境隔离和工作区指标门禁均通过。直接晋升组在单次成功和敏感记录的脱敏衍生经验上出现误晋升；另外两组没有出现误晋升或高价值记忆遗漏。

首次评分表把人工回执混入所有条件的正确性，会偏向分阶段人工门禁。复核后将核心决策正确性与机制合规分开，并明确“脱敏不等于已经具备晋升证据”。夹具、提示和模型输入没有变化，因此不重跑 Pilot。完整复核见 [`analysis/pilot-01.md`](analysis/pilot-01.md)。

## 正式实验门槛

Pilot 已通过协议门禁。正式矩阵冻结为 `5 个任务 x 3 个条件 x 3 次`，共 45 次；模型、推理强度、Codex CLI、夹具、提示和逐任务评分表在运行前固定记录。

macOS 是首轮主体实验平台。Win11 使用相同冻结协议进行后续复现；两个平台都完成有效正式矩阵后，文章才可以声称该晋升流程获得了本实验范围内的跨平台实测证据。

macOS 45 次正式矩阵已经完成，运行门禁全部通过。只读评分建议见 [`analysis/formal-macos-review.md`](analysis/formal-macos-review.md)；由于 Review 开始前没有记录真实时间，未写入正式评分或聚合。Win11 已完成 45 次正式矩阵、真实计时评分和脱敏聚合，结果为 `210/225`，文件位于 `data/`。

矩阵中断恢复时，只有成功退出、最终答案与用量存在、环境隔离有效、指标覆盖可靠且运行文件完整的目录才允许跳过。失败运行即使包含 `metadata.json` 也必须先隔离，随后使用相同实验槽位重跑；已经完成的有效样本不需要重复执行。

## 有效性限制

- 夹具是小型合成项目，不能代表所有团队的证据复杂度。
- 三种机制是可解释的对照条件，不覆盖所有记忆产品和 Agent 工具。
- 人工门禁质量取决于决策者获得的证据，不等于“有人点确认就一定正确”。
- 本实验验证晋升前的决策，不处理晋升后的冲突替代、降级和删除；这些属于下一篇的治理主题。
