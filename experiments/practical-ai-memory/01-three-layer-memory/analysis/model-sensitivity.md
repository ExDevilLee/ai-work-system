# 模型敏感性对照

日期：2026-07-17

## 对照范围

主实验使用：

- `gpt-5.6-sol`
- `medium`
- 每个任务和条件 3 次

探索性对照使用：

- `gpt-5.6-terra`
- `medium`
- 每个任务和条件 1 次

两组均使用 `codex-cli 0.144.1`、同一冻结夹具、同一提示、只读沙箱和独立新会话。探索性对照只有 `n=1`，不能用于稳定模型排名。

## 正确性

两个模型配置的所有运行均为满分。在当前简单、答案确定的任务中，没有观察到正确性差异。

## 工作区读取差异

主实验列使用 3 次中位数；探索性对照列使用单次值：

| 任务 | 条件 | `gpt-5.6-sol` | `gpt-5.6-terra` |
| --- | --- | ---: | ---: |
| 当前任务 | Baseline | 3,628 B | 4,164 B |
| 当前任务 | Layered | 575 B | 698 B |
| 稳定规则 | Baseline | 3,628 B | 3,628 B |
| 稳定规则 | Layered | 1,595 B | 1,193 B |
| 参考资料 | Baseline | 4,058 B | 4,171 B |

两个模型都表现出 Layered 读取项目文本较少，但具体数值和导航方式不同。

## 工具策略差异

- 当前任务中，主模型通常直接读取目标文件；探索性对照先列出文件，再读取目标文件。
- 探索性对照在 Layered 参考资料任务中额外读取了无关的 `article-review-checklist.md`。
- 主模型的参考资料任务会在“直接读索引和目标文件”与“额外检查入口说明”之间变化。

这说明目录结构只能提供导航信号，不能保证所有模型采用完全相同的最短读取路径。

## 输出成本差异

output token 没有呈现单一方向：

- 当前任务 Baseline：主模型中位数 444，对照模型 543。
- 稳定规则 Baseline：主模型中位数 833，对照模型 2,307。
- 稳定规则 Layered：主模型中位数 1,057，对照模型 492。
- 参考资料 Layered：主模型中位数 783，对照模型 2,726。

因此不能用“某模型整体更简洁”概括当前数据。模型、任务和文件组织之间存在交互，且单次对照可能受到生成波动影响。

## 面向读者的说明

文章需要明确写出实验使用的模型和推理强度。读者使用其他模型复现实验时，答案可能同样正确，但读取文件数量、是否先查看索引、是否读取无关资料、回答长度和耗时都可能不同。

可复现目标应优先关注：

1. 是否找到正确层级和来源。
2. 是否避免把临时观察当成稳定事实。
3. 是否在不读取无关层的情况下完成单一任务。

不应要求其他模型复现完全相同的调用次数或 token 数。

本阶段不生成模型对比图。探索性对照只有一次，用图形展示容易让单点差异看起来比实际更稳定。
