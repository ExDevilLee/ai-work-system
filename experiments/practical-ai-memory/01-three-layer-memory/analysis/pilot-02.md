## 主要观察

### 1. 分层减少了本次加载的项目文本

三个任务中，Layered 的工作区输出字节数分别比 Baseline 低约 84.2%、27.7% 和 62.8%。这说明按任务职责提供较小目标文件，在当前夹具中确实减少了返回给 Agent 的项目文本。

这仍然是单次试跑结果。正式实验完成前，只能表述为“协议已经能够测出差异”，不能写成普遍规律。

### 2. 分层不等于调用更少

参考资料任务中，Layered 先查看文件列表、`AGENTS.md` 和索引，再读取两个资料文件，共产生 5 次工作区调用；Baseline 读取混合记录后直接核对两个资料文件，共 3 次。

分层可能减少单次加载内容，却增加导航步骤。文章后续需要同时展示“读取文本量”和“调用次数”，不能只选择有利指标。

### 3. 混合记录也可以定向检索

稳定规则任务中，Baseline 使用 `rg` 筛选关键词，没有机械地读取整个混合文件。这否定了“混合文件必然每次全量进入上下文”的简单假设。

真实比较应允许 Agent 使用合理工具策略。三层结构的价值不能建立在刻意削弱 Baseline 上。

### 4. 耗时和全局 token 暂不能比较

Layered 的参考资料任务耗时 420.581 秒，是明显运行服务异常值。不同会话还会不稳定地读取全局技能，使 input token 出现与夹具无关的波动。

因此正式实验将正确性和 `workspace_output_bytes` 作为主要指标；工作区调用次数作为次要指标；耗时和 Codex CLI 全局 token 只保留为观察值，不用于单独支持结构优劣结论。

## 被保留的失败

Pilot 02 首次参考资料任务中，Baseline 的混合记录声明了两个资料路径，但文件并不存在，而 Layered 存在真实文件。Baseline 主动报告“路径真实性不确定”，暴露出夹具不等价。

这对运行的答案内容虽然为 8/8，但 `protocol_valid` 标记为 `false`，不进入条件比较。修正夹具后使用 `pilot-02b` 重跑，两个条件都成功核对真实资料文件。

## 正式实验门禁

协议可以进入正式重复实验，但需要冻结以下规则：

1. 使用当前 `pilot-02` 修正后夹具和三个独立任务。
2. 显式锁定 `gpt-5.6-sol`、只读沙箱和 ephemeral 新会话。
3. 每个任务、每个条件至少运行 3 次，共 18 次。
4. Baseline 与 Layered 交替执行，任务顺序轮换。
5. 每次检查 Layered 是否读取了任务明确排除的层。
6. 主要报告正确性、工作区输出字节和失败模式，不对小样本作统计显著性宣称。
7. 完成正式数据复核前不开始公开文章初稿。
