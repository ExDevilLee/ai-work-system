## 条件差异

### `append-only`

- 明确替代后仍同时保留增量导航和全量重建两个当前动作。
- 范围收窄后仍同时保留“全部平台 PNG”和“仅 Wiki-A PNG”，导致 Blog-B 与 Note-C 没有唯一动作。
- 紧急撤销后仍让自动公开与保持私有同时有效，无法选择安全动作。
- 原记录预设到期的临时规则可以自然结束，不受追加限制影响。

### `latest-wins`

- 明确替代、范围收窄、到期和紧急撤销均能采用日期最新的记录。
- 未解决冲突中直接采用较新的 180 秒候选结论，不等待控制变量复验或人工决定。

### `lifecycle-governed`

- 使用 `superseded`、`contested`、`expired` 和 `revoked` 区分非当前记录。
- 五类场景均保留历史来源，同时只让 `active` 记忆指导当前行动。
