## 分任务结果

| 任务 | 只追加 | 最新记录优先 | 生命周期治理 |
| --- | ---: | ---: | ---: |
| 明确替代 | 6/18 | 18/18 | 18/18 |
| 未解决冲突 | 15/15 | 3/15 | 15/15 |
| 时间到期 | 15/15 | 15/15 | 15/15 |
| 范围收窄 | 6/18 | 18/18 | 18/18 |
| 紧急撤销 | 9/18 | 18/18 | 18/18 |

只追加机制能够保留冲突，不会静默覆盖未解决结论，也能处理原记录预设的有效期；但旧规则没有明确退出当前行动的状态，因此在替代、范围收窄和撤销场景中持续产生多重有效动作。

最新记录优先可以低成本处理替代、到期、范围变化和撤销，却把“日期较新”误当成“证据已经解决冲突”。三次未解决冲突任务都直接采用 180 秒候选值，没有等待控制网络出口、服务配额和请求频率后复验。

生命周期治理在当前五类任务中全部命中冻结 rubric。`superseded`、`contested`、`expired` 和 `revoked` 让历史记录继续可审计，同时避免其继续指导当前行动。

## 当前可以支持的结论

1. 长期记忆仅保留历史而没有显式退出机制，会让已替代或已撤销规则继续污染当前行动。
2. 最新记录优先适合处理明确替代，但不能代替冲突识别和受控复验。
3. 预设有效期可以让临时规则自然退出，不一定依赖后续撤销记录。
4. 显式生命周期状态可以同时满足历史可审计和当前动作唯一性。
5. 紧急撤销需要恢复证据和明确批准，不能只依赖格式检查或自动恢复。

## 有效性限制

- 每个任务与条件只有 3 次重复，不支持稳定的耗时或 token 排名。
- 使用合成夹具，尚未覆盖真实长期记忆中的记录规模、并发写入和人工治理成本。
- Review 未盲化条件名称，45 次采用一次真实计时后的批量均摊方式。
- 结果只覆盖当前模型、推理强度、CLI 和 macOS 工具链。
- Win11 尚未复现，当前不能声称跨平台结论成立。
