## 发现的问题

### `append-only` 的停用语义不明确

在 `explicit-supersession` 中，模型认为新决定不能停用旧规则，因此报告没有唯一动作；但在 `scope-narrowing` 和 `emergency-revocation` 中，它又让新决定停止了旧动作，同时保留旧记录的历史状态。

这不是稳定的条件差异，而是条件文本允许两种解释：

- 只禁止修改历史记录；
- 同时禁止停止历史规则的当前效力。

### `latest-wins` 仍保留审批语义

在 `unresolved-conflict` 中，模型没有直接采用日期较新的 180 秒记录，而是继续考虑候选结论尚未批准。这使该条件没有稳定代表朴素的“最新记录直接覆盖”。

## Pilot 02 修订

- `append-only` 明确为：无预设到期条件的旧规则持续有效；后续决定只能追加，不能停用旧动作；新旧冲突时报告没有唯一动作。
- `latest-wins` 明确为：日期最新的记录直接成为当前可执行结论，不等待批准或冲突解决；只有最新记录本身明确暂停时例外。
- `lifecycle-governed` 不修改。
- 共享记录、任务提示、冻结答案和 rubric 不修改。
- Pilot 02 使用独立运行标签，不复用 Pilot 01。

只有 Pilot 02 同时通过执行门禁和条件可区分性 Review，才考虑冻结正式矩阵。
