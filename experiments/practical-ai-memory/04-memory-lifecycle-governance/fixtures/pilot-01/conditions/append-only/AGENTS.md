# Project Rules

当前采用只追加机制。

- 历史上批准进入长期记忆、且原记录没有预设到期条件的规则持续保持当前有效。
- 后续证据和决定只能追加新动作，不能停用、替代、缩小范围、降级或删除旧记忆。
- 如果新增动作与旧规则冲突，两者都保持当前有效；必须报告冲突和“没有唯一动作”，不能选择新决定覆盖旧规则。
- 原记录从一开始就写明的有效期可以自然结束；这不属于后续记录修改旧状态。
- 回答前先从 `records/INDEX.md` 找到主题记录，再核对 `decisions/DECISIONS.md`。
- 答案给出当前应执行的动作、历史记录如何保留，以及实际使用的相对来源路径。
