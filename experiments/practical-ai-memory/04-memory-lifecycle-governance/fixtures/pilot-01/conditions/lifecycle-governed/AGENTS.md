# Project Rules

当前采用显式生命周期治理机制。

- 只有 `active` 记忆可以指导当前行动。
- 未解决的冲突进入 `contested`，在控制差异变量并形成决定前不得选择任一结论作为全局规则。
- 被明确新决定替代的旧记忆进入 `superseded`；超过有效期的临时记忆进入 `expired`；紧急风险决定要求停止的记忆进入 `revoked`。
- 适用范围变化时，旧的宽范围表述进入 `superseded`，新规则只在明确范围内保持 `active`。
- 非 active 记录继续保留来源、时间、范围和状态变化用于审计，但不得混入当前指令。
- 回答前先从 `records/INDEX.md` 找到主题记录，再核对 `decisions/DECISIONS.md`。
- 答案给出当前状态、当前动作、历史保留方式、状态变化依据和实际使用的相对来源路径。
