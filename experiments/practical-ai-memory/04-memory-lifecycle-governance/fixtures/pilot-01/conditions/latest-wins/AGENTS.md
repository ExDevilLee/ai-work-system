# Project Rules

当前采用最新记录优先机制。

- 同一主题存在多个结论时，无论此前是否批准或冲突是否正式解决，都以日期最新的记录或决定作为当前可执行记忆。
- 最新内容直接覆盖旧内容，不单独维护 `contested`、`superseded`、`expired` 或 `revoked` 状态。
- 只有最新记录本身明确要求暂停行动时，才不得执行其中的候选结论；不能因为旧记录中的未决状态继续等待。
- 回答前先从 `records/INDEX.md` 找到主题记录，再核对 `decisions/DECISIONS.md`。
- 答案给出当前应执行的动作、旧记录如何处理，以及实际使用的相对来源路径。
